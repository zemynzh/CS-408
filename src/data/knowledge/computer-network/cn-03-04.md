# 后退N帧协议 (GBN)

## 1. 基本原理
- 发送窗口 $W_T > 1$，接收窗口 $W_R = 1$。
- 采用**累积确认** (Cumulative ACK)：ACK n 表示已收到第 n 号帧及之前的所有帧。
- 乱序到达：接收方丢弃所有非期望序号的帧。

## 2. 发送窗口大小限制
对于 $n$ 位编号：
$$1 \le W_T \le 2^n - 1$$
- 若 $W_T > 2^n - 1$，会产生序号回绕，无法区分是新帧还是旧帧。

## 3. 性能
- 在误码率较高时，性能显著下降，因为需要重传所有未确认的帧。
- 信道利用率提高，适合高带宽、低误码率环境。
