# 计算机网络体系结构

## 1. 体系结构的概念
计算机网络体系结构是计算机网络各层及其协议的集合。体系结构是抽象的，而实现是具体的。

## 2. ISO/OSI 七层参考模型
从下到上依次为：
1. **物理层 (Physical)**：比特流传输。
2. **数据链路层 (Data Link)**：成帧、差错控制、流量控制。
3. **网络层 (Network)**：路由选择、拥塞控制、网际互连。
4. **会话层 (Session)**：管理主机间的会话进程。
5. **表示层 (Presentation)**：处理数据格式、加密、压缩。
6. **应用层 (Application)**：为用户提供服务接口。

## 3. TCP/IP 四层模型
1. **网络接口层**：对应 OSI 物理层和数据链路层。
2. **网际层 (IP)**：对应 OSI 网络层，核心是 IP 协议。
3. **传输层 (TCP/UDP)**：对应 OSI 传输层。
4. **应用层**：对应 OSI 会话层、表示层、应用层。

## 4. 五层参考模型（综合模型）
这是学习和 408 考研中最常用的模型：
1. **物理层**
2. **数据链路层**
3. **网络层**
4. **传输层**
5. **应用层**

## 5. 重要结论
- **协议**是“水平”的，控制对等实体之间通信的规则。
- **服务**是“垂直”的，下层通过服务访问点（SAP）为上层提供服务。
- 协议必须包含三个要素：**语法、语义、同步**（时序）。
