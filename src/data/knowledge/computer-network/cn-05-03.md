# TCP 连接管理 (握手挥手)

## 1. 三次握手 (连接建立)
1. **SYN (seq=x)**：客户机发送 SYN，处于 SYN_SENT。
2. **SYN+ACK (seq=y, ack=x+1)**：服务器响应，处于 SYN_RCVD。
3. **ACK (seq=x+1, ack=y+1)**：客户机确认，处于 ESTABLISHED。服务器收到后也进入 ESTABLISHED。

## 2. 四次挥手 (连接释放)
1. **FIN (seq=u)**：主动方发送 FIN，进入 FIN_WAIT_1。
2. **ACK (ack=u+1)**：被动方确认，进入 CLOSE_WAIT（半关闭）。主动方进入 FIN_WAIT_2。
3. **FIN+ACK (seq=v, ack=u+1)**：被动方发送 FIN，进入 LAST_ACK。
4. **ACK (ack=v+1)**：主动方确认，进入 **TIME_WAIT**。经过 **2MSL** 后关闭。

## 3. TIME_WAIT 的作用
- 确保最后一个 ACK 成功到达。
- 让本连接产生的重复分量在网络中消失，避免干扰新连接。
