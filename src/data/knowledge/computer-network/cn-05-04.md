# TCP 可靠传输与流量控制

## 1. 可靠传输机制
- **校验**：通过伪首部校验和检测错误。
- **序号与确认**：累计确认机制。
- **超时重传**：发送方根据 RTT 动态计算 RTO（超时重传时间）。
- **快速重传**：连续收到 3 个重复 ACK，立即重传。

## 2. 流量控制 (Flow Control)
- **目的**：匹配发送方的发送速率与接收方的处理速率。
- **手段**：**滑动窗口**。
- **机制**：
  - 接收方在 ACK 报文段的“窗口”字段宣告接收窗口大小 (rwnd)。
  - 发送方的发送窗口 $swnd = \min(cwnd, rwnd)$。
- **零窗口死锁**：接收方发送零窗口 ACK 后，后续通知窗口非零的 ACK 丢失。
  - **解决方法**：持久计时器 (Persistence Timer)，发送探测报文。
