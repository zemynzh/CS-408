# 第四章 文件管理 (4.2) —— 文件系统实现

## 1. 文件分配方式 (Allocation Methods)
1. **连续分配**:
    - **原理**: 文件占据相邻的物理块。
    - **优点**: 顺序访问快，查找容易。
    - **缺点**: 易产生碎片，文件大小难以扩充。
2. **链接分配**:
    - **隐式链接**: 每个块存下一个块的指针（离散）。
        - **缺点**: 只能顺序访问，若一个块坏了则后续全部失效。
    - **显式链接 (FAT)**: 所有指针存放在 **文件分配表 (FAT)**。
        - **FAT**: 全局表，存放在内存中（大幅提高访问速度）。
3. **索引分配**:
    - **原理**: 给每个文件分配一个 **索引块**，存放该文件所有盘块的索引。
    - **多级索引**: 若一个索引块放不下，可建立二级、三级索引。
    - **计算 (408 常考)**: 给定块大小（如 1KB）、地址项大小（如 4B），计算三级索引支持的最大文件。
        - $N = 1KB/4B = 256$
        - 最大块数 = $N^3$

## 2. 空闲存储空间管理
1. **空闲表法**: 记录起始块号和块数（连续分配常用）。
2. **空闲链表法**: 以块或区为单位串联（离散分配常用）。
3. **位示图法 (Bitmap)**:
    - **原理**: 用 0/1 位表示空闲/已用。
    - **计算**: 磁盘容量/块大小 = 位数。
4. **成组链接法**: 
    - **原理**: Unix 系统采用，通过栈结构管理空闲块。第一个块存放下一组 100 个块的地址，依此类推。
    - **优点**: 效率极高。
