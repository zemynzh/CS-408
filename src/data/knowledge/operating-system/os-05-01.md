# 第五章 设备管理 (5.1) —— I/O 控制器与控制方式

## 1. I/O 控制器
- **作用**: 负责 CPU 与 I/O 设备之间的通信。
- **组成**:
    - **状态寄存器**: 指示设备是否忙碌、空闲。
    - **控制寄存器**: 接收 CPU 指令。
    - **数据寄存器**: 暂存数据。
    - **地址译码器**: 识别 CPU 地址总线。

## 2. I/O 控制方式 (408 重点)
1. **程序直接控制方式 (轮询)**:
    - **原理**: CPU 循环检查状态寄存器。
    - **缺点**: CPU 与 I/O 串行工作，利用率极低。
2. **中断驱动方式**:
    - **原理**: CPU 发出 I/O 请求后继续执行，I/O 完成时向 CPU 发出中断。
    - **优点**: CPU 与 I/O 并行工作。
    - **缺点**: 以 **字节** 为单位频繁中断，开销大。
3. **DMA (直接存储器存取)**:
    - **原理**: 在 I/O 设备与内存之间开辟通路，由 DMA 控制器管理。
    - **特点**: 以 **块** 为单位传输，仅在块传输结束时发出中断。
    - **优点**: CPU 参与少，效率高。
4. **I/O 通道方式**:
    - **原理**: 通道是专门处理 I/O 的 **协处理器**。
    - **特点**: 独立运行通道程序，执行一系列 I/O 任务。
    - **优点**: 极大地减轻了 CPU 负担。
