# 第二章 进程管理 (2.3) —— 进程同步与互斥

## 1. 临界区 (Critical Section)
- **临界资源**: 只能在一个时刻由一个进程访问的资源（如打印机）。
- **四个原则**: 空闲让进、忙则等待、有限等待、让权等待。

## 2. 信号量 (Semaphore) PV 操作
- **定义**: 信号量 $S$ 是一个整型量，表示某种资源的数量。
- **P 操作 (Wait)**: $S = S - 1$；若 $S < 0$，进程进入等待队列。
- **V 操作 (Signal)**: $S = S + 1$；若 $S \le 0$，唤醒等待队列中的一个进程。

**经典案例**:
1. **进程互斥**: 设置初值为 1 的信号量 `mutex`。
    - `P(mutex)` -> 临界区 -> `V(mutex)`。
2. **进程同步**: 设置初值为 0 的信号量 `S`。
    - 进程 A 执行完后 `V(S)`；进程 B 在开始执行前 `P(S)`（B 等待 A）。
3. **前驱图**: A 完成后执行 B, C，则 A 执行完后 `V(Sb), V(Sc)`，B 开始前 `P(Sb)`，C 开始前 `P(Sc)`。

## 3. 经典同步问题 (408 常考)
1. **生产者-消费者**:
    - `empty`: 缓冲区空位（初值为 $N$）。
    - `full`: 缓冲区产品（初值为 0）。
    - `mutex`: 互斥信号量（初值为 1）。
2. **读者-写者**:
    - 读-读不互斥，读-写、写-写互斥。
    - **读优先**: 依靠 `readcount` 记录读者数量，第一个读者加锁，最后一个读者解锁。
3. **哲学家进餐**: 
    - 核心在于避免死锁（一次拿两把叉子或规定取叉顺序）。
4. **吸烟者问题**:
    - 多级生产者-多级消费者同步。
    - 资源轮流供应（通过信号量数组或标志位）。
