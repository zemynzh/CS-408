# 第五章 设备管理 (5.2) —— I/O 软件层次结构

## 1. I/O 软件设计目标
- **设备独立性 (Device Independence)**: 用户程序不依赖于具体的硬件。
- **统一命名**: 每个设备都有一个唯一的逻辑名称。
- **错误处理**: 在较低层处理（如重试）。

## 2. I/O 软件各层职责 (从高到低)
1. **用户空间 I/O 软件**:
    - **职责**: 发出 I/O 请求，进行格式化。
    - **典型**: 标准 I/O 库（`printf`, `scanf`）、**SPOOLing 系统**。
2. **设备独立性软件 (Device-Independent OS Software)**:
    - **职责**: 映射逻辑名称、分配设备、**缓冲管理**、设备保护。
3. **设备驱动程序 (Device Driver)**:
    - **职责**: 将抽象的 I/O 请求转为具体控制指令（如写磁盘扇区、发控制字）。
    - **特点**: 与硬件强相关。
4. **中断处理程序 (Interrupt Handler)**:
    - **职责**: 处理 I/O 完成后的中断，唤醒被阻塞的进程。
    - **特点**: 内核中运行。
5. **硬件 (Hardware)**:
    - 执行物理 I/O 操作。
