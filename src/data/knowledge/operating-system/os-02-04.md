# 第二章 进程管理 (2.4) —— 死锁

## 1. 死锁定义 (Deadlock)
- **定义**: 多个进程互相等待对方持有的资源而都无法继续运行的现象。
- **必要条件**:
    1. **互斥 (Mutual Exclusion)**: 资源只能由一个进程占用。
    2. **不剥夺 (No Preemption)**: 资源只能由该进程自己释放。
    3. **请求与保持 (Hold and Wait)**: 进程已占有一个资源，且在申请另一个资源。
    4. **循环等待 (Circular Wait)**: 进程间形成环路。

## 2. 死锁的处理策略
1. **死锁预防 (Prevention)**: 破坏四个必要条件之一（开销大，效率低）。
2. **死锁避免 (Avoidance)**: 防止进入不安全状态（如 **银行家算法**）。
3. **死锁检测 (Detection)**: 定期检查系统是否发生死锁（利用 **资源分配图**）。
4. **死锁解除 (Recovery)**: 撤销死锁进程或剥夺其资源。

## 3. 银行家算法 (Banker's Algorithm)
- **核心思想**: 每一个请求资源时，先模拟分配，检查系统是否处于 **安全状态**。
- **数据结构**:
    - `Max`: 进程对各类资源的最大需求。
    - `Allocation`: 已分配给各进程的资源数。
    - `Need = Max - Allocation`: 进程还需要的资源数。
    - `Available`: 系统现有的空闲资源。

**安全序列**: 如果存在一个序列 $P_1, P_2, ..., P_n$，按照该序列分配资源，每个进程都能顺利完成并释放资源。

## 4. 资源分配图
- **节点**: 进程、资源类。
- **边**:
    - **分配边 (R -> P)**: 指向进程，表示已分配。
    - **请求边 (P -> R)**: 指向资源，表示进程在等待。
- **死锁定理**: 如果资源分配图可以简化（消除非阻塞进程的所有边），则系统无死锁。
