# 第一章 操作系统概述 (1.3) —— 操作系统运行环境

## 1. 处理器运行模式
1. **特权指令 (Privileged Instruction)**: 只有在内核态下才能执行。如 I/O 指令、设置时钟、清内存、中断管理。
2. **非特权指令**: 用户程序只能使用非特权指令。
3. **内核态 (Kernel Mode)**: CPU 具有最高权限，可以执行所有指令，访问所有软硬件资源。
4. **用户态 (User Mode)**: 限制在一定范围内访问内存，禁止使用特权指令。

**状态切换**:
- **用户态 -> 内核态**: 唯一途径是 **中断 (Interrupt)** 或 **异常 (Exception)**。
- **内核态 -> 用户态**: 通过设置 PSW（程序状态字）的标志位。

## 2. 中断与异常
- **中断 (外中断)**: 源自 CPU 外部（如时钟、键盘、I/O 结束）。
- **异常 (内中断/陷阱)**: 源自 CPU 内部（如越界、溢出、除零、系统调用指令）。
    - **故障 (Fault)**: 可以被修复（如缺页异常）。
    - **中止 (Abort)**: 无法恢复。

## 3. 系统调用 (System Call)
系统调用是操作系统提供给应用程序的接口。
- **作用**: 让用户程序能够请求操作系统的特权服务。
- **执行过程**:
    1. 传递系统调用参数。
    2. 执行 **访管指令 (Trap 指令/自陷指令)**，CPU 进入内核态。
    3. 操作系统内核根据调用号跳转到相应的系统调用处理子程序。
    4. 完成后返回用户态。
    
**注意**: 访管指令是非特权指令，但它会导致 CPU 进入内核态。系统调用本身是在内核态运行的。
