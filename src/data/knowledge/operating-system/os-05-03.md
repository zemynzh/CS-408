# 第五章 设备管理 (5.3) —— 缓冲与 SPOOLing

## 1. 缓冲技术 (Buffering)
- **目的**: 缓解 CPU 与 I/O 设备之间的 **速度不匹配**，减少 CPU 中断频率。
- **单缓冲**:
    - **原理**: 设备与内存之间开辟一个缓冲区。
    - **耗时**: $T = \max(C, T) + M$（$C$: CPU 处理，$T$: 设备输入，$M$: 缓冲区传向用户区）。
- **双缓冲**:
    - **原理**: 开辟两个缓冲区。
    - **耗时**: $T = \max(C, T)$。
- **循环缓冲、缓冲池**: 进一步提高效率。

## 2. SPOOLing 技术 (假脱机)
- **原理**: 在外存（磁盘）上开辟 **输入井** 和 **输出井**，将慢速 I/O 设备模拟为共享设备。
- **特点**:
    1. **虚拟性**: 将独占设备模拟为共享设备。
    2. **时分复用**: 多个进程可以同时将数据存入磁盘输出井。
    3. **加快响应**: 进程提交 I/O 后立即返回，由 SPOOLing 守护进程在后台慢慢处理。

## 3. 设备分配 (Device Allocation)
1. **独占设备**: 进程独占直到释放（如磁带机）。
2. **共享设备**: 多个进程同时访问（如磁盘）。
3. **虚拟设备**: 通过 SPOOLing 将独占变为共享。

## 4. 故障处理
- **重试、纠错码**。
- **坏块管理**: 维护坏块映射表。
