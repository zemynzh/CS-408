# 第二章 进程管理 (2.1) —— 进程与线程

## 1. 进程概念 (Process)
- **定义**: 一个正在执行的程序的实例，是资源分配的基本单位。
- **PCB (Process Control Block)**: 进程存在的唯一标志。包含进程标识、状态、优先级、程序计数器、上下文。

## 2. 进程状态转换
1. **就绪 (Ready)**: 已分配 CPU 以外的所有资源，等待调度。
2. **执行 (Running)**: 正在占用 CPU 执行程序。
3. **阻塞 (Waiting/Blocked)**: 因等待某种事件（如 I/O 结束、信号量）而暂停。
4. **创建 (New)**: 正在申请 PCB。
5. **终止 (Exit)**: 正在释放 PCB。

**转换关系**:
- **就绪 -> 执行**: 进程调度程序选中。
- **执行 -> 就绪**: 时间片用完或被更高优先级抢占。
- **执行 -> 阻塞**: 进程请求 I/O、等待事件发生。
- **阻塞 -> 就绪**: I/O 完成或等待事件已发生。

## 3. 进程控制
- **创建、撤销、阻塞、唤醒**: 这些由内核中的 **原语 (Primitve)** 完成。原语是具有原子性的指令序列（通过关中断、开中断实现）。

## 4. 线程概念 (Thread)
- **定义**: 处理器调度的基本单位。
- **与进程的区别**:
    - 进程是 **资源分配** 单位；线程是 **调度** 单位。
    - 一个进程可以包含多个线程，共享进程的资源（代码、数据、堆）。
    - 线程拥有独立的 **栈** 和 **程序计数器 (PC)**。

## 5. 多线程模型
- **一对一**: 用户线程与内核线程一一对应。
- **多对一**: 多个用户线程对应一个内核线程（开销小，但一个线程阻塞则全进程阻塞）。
- **多对多**: 多个用户线程对应多个内核线程。
