# 第三章 内存管理 (3.3) —— 虚拟内存管理基础

## 1. 虚拟内存定义 (Virtual Memory)
- **局部性原理 (Locality)**:
    - **时间局部性**: 指令被重复执行，数据被重复访问。
    - **空间局部性**: 访问了某个存储单元，不久后也访问其附近的存储单元。
- **虚拟存储器的三个特征**:
    1. **多次性**: 分多次装入内存。
    2. **对换性**: 允许换出暂时不用的。
    3. **虚拟性**: 逻辑上扩充了内存容量。

## 2. 请求分页管理方式 (Demand Paging)
- **原理**: 运行时只装入当前需要的页面，发生 **缺页异常 (Page Fault)** 时再调入。
- **页表项 (Page Table Entry)**:
    - `页号 | 状态位 | 访问字段 | 修改位 | 外存地址`。
    - **状态位**: 页面是否在内存。
    - **修改位**: 是否被写过，若为 1 则换出时需写回外存。
- **缺页异常处理流程**:
    1. 查找页表，状态位为 0。
    2. 保存 CPU 现场。
    3. 缺页中断响应，查询外存获取页面。
    4. 将页面调入内存。
    5. 修改页表，返回用户态。

## 3. 内存分配策略
1. **固定分配、局部置换**: 给每个进程固定数目的页框。
2. **可变分配、局部置换**: 随时间动态调整，但只置换自己的页面。
3. **可变分配、全局置换**: 置换空闲页框或其他进程的页框（常用）。
