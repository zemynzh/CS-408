# 第二章 进程管理 (2.2) —— 处理机调度算法

## 1. 调度层次
1. **高级调度 (作业调度)**: 将磁盘作业调入内存，分配资源并创建进程（从外存到内存）。
2. **中级调度 (内存调度)**: 将暂时不运行的进程调出内存（挂起状态），以提高内存利用率（内存与外存对换）。
3. **低级调度 (进程调度)**: 从就绪队列中选择一个进程并分配 CPU（最频繁）。

## 2. 调度性能指标
- **周转时间**: $T = 完成时间 - 提交时间$
- **平均周转时间**: $T_{avg} = \frac{\sum T_i}{n}$
- **带权周转时间**: $W = \frac{T}{服务时间} \ge 1$
- **平均带权周转时间**: $W_{avg} = \frac{\sum W_i}{n}$
- **响应比**: $R_p = \frac{等待时间 + 服务时间}{服务时间}$

## 3. 常见调度算法
1. **先来先服务 (FCFS)**: 对长作业有利，对短作业不利。
2. **短作业优先 (SJF/SPF)**: 理论上平均周转时间最短。
    - **非抢占式**: 当前执行完再选。
    - **抢占式 (SRT)**: 随时选剩余时间最短的。
3. **时间片轮转 (RR)**: 用于分时系统。时间片过大退化为 FCFS；时间片过小增加切换开销。
4. **优先级调度 (Priority Scheduling)**:
    - **静态优先级/动态优先级**。
    - **抢占式/非抢占式**。
5. **多级反馈队列调度 (MLFQ)**: 
    - 结合了 FCFS, RR, SJF。
    - 多个就绪队列，优先级从高到低，时间片从小到大。
    - 新进程入队列 1，运行完后入下一级队列（如果未结束）。
    - 队列 1 运行空了再运行队列 2。
    - **优点**: 终端用户响应快，短作业也快。
