# 第三章 内存管理 (3.4) —— 页面置换算法

## 1. 最佳置换算法 (OPT - Optimal)
- **原理**: 选出未来最长时间内不再使用的页面。
- **优点**: 缺页率最低。
- **缺点**: 无法实现（需要预知未来）。

## 2. 先进先出置换算法 (FIFO)
- **原理**: 置换在内存中驻留时间最长的页面。
- **缺点**: 
    - 缺页率高。
    - **Belady 异常**: 分配的页框数增加，缺页率反而上升（只有 FIFO 有此现象）。

## 3. 最近最久未使用算法 (LRU - Least Recently Used)
- **原理**: 置换最近最久未使用的页面（利用局部性原理）。
- **实现**: 堆栈或计数器。
- **性能**: 较好，接近 OPT。
- **开销**: 需要硬件支持，开销大。

## 4. 时钟置换算法 (CLOCK / NRU)
- **原理**: 页面设置一个访问位（Use Bit）。
- **简单 CLOCK 算法**:
    1. 页面访问位设为 1。
    2. 需要置换时，指针顺时针扫描，遇到 1 变 0，遇到 0 则置换。
- **改进 CLOCK 算法**: 增加修改位（Modified Bit）。
    - 扫描顺序: 
        1. (0, 0): 未访问且未修改（优先）。
        2. (0, 1): 未访问但已修改。
        3. (1, 0): 已访问但未修改。
        4. (1, 1): 已访问且已修改。

## 5. 抖动 (Thrashing)
- **定义**: 进程频繁地在内存与外存之间交换页面，导致 CPU 大部分时间用于页面调度。
- **原因**: 分配给进程的物理块太少。
- **解决**: 使用 **工作集 (Working Set)**。
