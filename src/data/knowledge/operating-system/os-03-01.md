# 第三章 内存管理 (3.1) —— 内存管理基础

## 1. 链接与装入
1. **链接 (Linking)**: 将多个模块及库函数合成一个可执行模块。
    - **静态链接**: 执行前链接。
    - **装入时动态链接**: 装入内存时边装入边链接。
    - **运行时动态链接**: 执行到某模块时再链接。
2. **装入 (Loading)**: 将可执行模块装入内存。
    - **绝对装入**: 编译时确定物理地址（单道程序）。
    - **可重定位装入 (静态重定位)**: 装入时完成逻辑到物理地址转换（多道程序）。
    - **动态运行时装入 (动态重定位)**: 程序运行时再执行转换，需要 **重定位寄存器**。

## 2. 连续分配管理方式
1. **单一连续分配**: 内存分为系统区和用户区，用户区仅一道程序。
2. **固定分区分配**: 内存分为若干个固定大小的分区（易产生 **内部碎片**）。
3. **动态分区分配**: 根据进程需要建立分区（易产生 **外部碎片**）。
    - **首次适应 (First Fit)**: 从头查找，找到第一个满足的分区。
    - **最佳适应 (Best Fit)**: 选最小且能满足的分区（产生碎多碎片）。
    - **最坏适应 (Worst Fit)**: 选最大且能满足的分区。
    - **邻近适应 (Next Fit)**: 从上次查找结束位置开始查找。

## 3. 基本分页管理方式 (Non-contiguous)
- **页 (Page)**: 逻辑地址空间划分（例如 4KB）。
- **页框 (Frame/Page Frame)**: 物理内存划分。
- **页表**: 保存页号到页框号的映射。
- **地址结构**: `页号 | 页内偏移`。
    - `页号 = 逻辑地址 / 页面大小`
    - `页内偏移 = 逻辑地址 % 页面大小`
- **逻辑地址转物理地址**: 物理地址 = 页框号 * 页面大小 + 页内偏移。
