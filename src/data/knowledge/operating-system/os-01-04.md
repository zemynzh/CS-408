# 第一章 操作系统概述 (1.4) —— 操作系统结构

## 1. 宏内核 (Monolithic Kernel)
- **特点**: 将 OS 的所有核心部分（进程、内存、设备、文件）都放在一个巨大的内核空间。
- **优点**: 高效（各模块间直接函数调用）。
- **缺点**: 结构混乱、不灵活、一个模块故障可能导致整个内核崩溃（如 Linux, UNIX, Windows 早前版本）。

## 2. 微内核 (Microkernel)
- **特点**: 将 OS 的核心服务（如中断、进程间通信 IPC）保留在内核中，而将文件系统、设备驱动等移到用户态。
- **优点**: 灵活、可靠（一个服务崩溃不影响其他部分）。
- **缺点**: 低效（频繁的内核态与用户态切换，频繁的 IPC）。

## 3. 分层结构 (Layered Structure)
- **特点**: 每层只能调用其下层的功能。
- **优点**: 易于调试和维护。
- **缺点**: 定义各层的职责较难，效率较低。

## 4. 模块化 (Modular)
- **特点**: 内核有一组核心组件，其他功能以可加载模块的形式实现（类似于面向对象）。
- **优点**: 内核非常紧凑且灵活（如现代 Linux 驱动加载）。

## 5. 外核 (Exokernel)
- **特点**: 内核只负责保护资源，不负责管理，将管理权限下放给应用程序。
