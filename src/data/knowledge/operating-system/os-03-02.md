# 第三章 内存管理 (3.2) —— 分页、分段与段页式

## 1. 分页管理 (Paging)
- **快表 (TLB)**: 存放在 CPU 缓存（Cache）中的高速缓冲页表。
    - **原理**: 局部性原理。
    - **查找**: 同时进行页表查找和 TLB 查找（减少内存访问次数）。
- **二级页表**: 
    - 解决页表过大的问题。将页表分页存放在内存。
    - **逻辑地址结构**: `外层页号 | 内层页号 | 页内偏移`。

## 2. 分段管理 (Segmentation)
- **分段原理**: 将进程的逻辑地址空间分为若干个逻辑段（如代码段、数据段、堆栈段）。
- **逻辑地址结构**: `段号 (S) | 段内偏移 (W)`。
- **段表**: `段号 | 段长 | 基址`。
- **地址转换**:
    1. 检查段号 $S$ 是否越界。
    2. 检查段内偏移 $W$ 是否超过段长。
    3. 物理地址 = 基址 + 段内偏移。

## 3. 段页式管理 (Segment-Paging)
- **原理**: 结合了分页和分段。
    - 先将进程分段（逻辑意义）。
    - 再将每个段分页（物理存储）。
- **逻辑地址结构**: `段号 (S) | 页号 (P) | 页内偏移 (W)`。
- **内存访问**:
    1. 查找段表（内存中）。
    2. 查找页表（内存中）。
    3. 访问物理单元。
    - 共计 **3 次内存访问**（不考虑 TLB）。
