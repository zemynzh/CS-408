# 指令流水线 (co-05-05)

## 1. 流水线的基本概念
- **阶段划分**：通常分为 IF (取指), ID (译码), EX (执行), MEM (访存), WB (写回)。

## 2. 流水线冲突 (Hazards)
1. **结构冲突 (Structural)**：多个指令同时抢占同一硬件资源 (如同时访问存储器)。解决：增加资源。
2. **数据冲突 (Data)**：当前指令依赖前一条指令尚未写回的结果。
  - **分类**：RAW (写后读)、WAR (读后写)、WAW (写后写)。
  - **解决**：数据旁路/转发 (Forwarding)、暂停流水线 (Stall)、编译器重排。
3. **控制冲突 (Control)**：由跳转 (Jump) 指令导致下一条指令地址不确定。
  - **解决**：分支预测 (Static/Dynamic)、延迟槽 (Delayed Slot)。

## 3. 性能计算 (重点)
- **吞吐率 ($TP$)**：单位时间内执行指令数。
  - $TP = \frac{n}{(k + n - 1)\Delta t}$ ($n$ 条指令, $k$ 段流水, $\Delta t$ 周期)。
- **加速比 ($S$)**：不使用流水线与使用流水线的时间之比。
  - $S = \frac{nk}{k + n - 1}$
- **效率 ($E$)**：流水线满载程度。
  - $E = \frac{n}{k + n - 1}$
- **实际执行时间**：$t = (k + n - 1) \Delta t$。
