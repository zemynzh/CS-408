# DMA 方式 (co-07-04)

## 1. DMA 基本原理
- **DMA (Direct Memory Access)**：外设直接与内存交换数据，不经过 CPU。
- **特点**：适合高速数据传输，块传输。
- **控制权**：由 **DMA 控制器 (DMAC)** 接管总线控制权。

## 2. DMA 传输过程 (重点)
1. **预处理**：CPU 初始化 DMAC (设置主存地址、字计数、传送方向)。
2. **数据传送**：
  - 设备向 DMAC 发请求。
  - DMAC 向 CPU 发总线请求 (BR)。
  - CPU 响应，交出总线所有权。
  - **数据直接在内存和设备间传送**。
3. **后处理**：传送完成后，DMAC 向 CPU 发中断信号，CPU 检查传输是否正确。

## 3. 总线占用方式
- **停止 CPU 访问**：传送期间 CPU 挂起。
- **周期挪用 (Cycle Stealing)**：DMA 挪用 CPU 不访问内存的周期。
- **交替访问**：CPU 和 DMA 交替使用周期 (需总线同步)。

## 4. 中断 vs DMA
- **中断**：指令执行结束响应；用于低速设备；由程序控制。
- **DMA**：机器周期结束响应 (存取周期)；用于高速设备；由硬件控制。
