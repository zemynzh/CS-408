# CISC vs RISC (co-04-03)

## 1. CISC (Complex Instruction Set Computer)
- **特点**：
  - 指令丰富、复杂、不等长。
  - 访存指令不限，寻址方式多。
  - 寄存器少。
  - 大多采用**微程序控制器**。
- **典型架构**：x86。

## 2. RISC (Reduced Instruction Set Computer)
- **特点**：
  - 指令精简、等长 (通常一个字长)。
  - 仅 Load/Store 指令访存。
  - 寻址方式少。
  - 寄存器多。
  - 采用**硬布线控制器**。
  - 极适合**流水线**执行。
- **典型架构**：ARM、MIPS、RISC-V。

## 3. 对比总结
| 特性 | CISC | RISC |
| :--- | :--- | :--- |
| 指令条数 | 多 (几百条) | 少 (几十条) |
| 指令长度 | 不固定 | 固定 |
| 访存指令 | 任意 | 仅 Load/Store |
| 控制方式 | 微程序 | 硬布线 |
| 流水线 | 不易优化 | 极易优化 |
