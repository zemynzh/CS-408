# 2.3 链表 (Linked List)

### 1. 单链表 (Singly Linked List)
通过一组任意的存储单元存储线性表中的数据元素，每个结点包含：数据域 `data` 和指针域 `next`。

#### 特点
- **非随机存取**：查找特定位序元素需从头遍历，$O(n)$。
- **插入删除方便**：只需修改指针，$O(1)$（已知前驱结点时）。

### 2. 双链表 (Doubly Linked List)
每个结点增加一个指向前驱的指针 `prior`。
- **插入/删除**：需要修改 4 个指针。
- **优点**：可以双向遍历，查找前驱结点时间复杂度为 $O(1)$。

### 3. 循环链表 (Circular Linked List)
- **循环单链表**：尾结点的 `next` 指向头结点。判断尾结点条件：`p->next == head`。
- **循环双链表**：头结点的 `prior` 指向尾结点，尾结点的 `next` 指向头结点。

### 4. 静态链表 (Static Linked List)
借助数组实现的链表。每个结点由 `data` 和 `next`（数组下标/游标）组成。
- 适用于不支持指针的语言。

### 5. 顺序表 vs 链表 (常考对比)
| 比较项目 | 顺序表 (顺序存储) | 链表 (链式存储) |
| :--- | :--- | :--- |
| **存取方式** | 随机存取 $O(1)$ | 顺序存取 $O(n)$ |
| **逻辑/物理关系** | 物理相邻 | 指针表示 |
| **插入/删除** | 需移动元素 $O(n)$ | 需找位置 $O(n)$，修改指针 $O(1)$ |
| **存储密度** | 高 (1.0) | 低 (需存指针) |

> **提示**：头结点的作用是统一空表和非空表的操作。
