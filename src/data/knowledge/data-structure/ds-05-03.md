# ds-05-03: 二叉树的存储结构

## 1. 顺序存储结构
使用一组地址连续的存储单元（如一维数组）自上而下、自左至右存储完全二叉树上的结点。
- **对应关系**：对于编号为 $i$ 的结点，其左孩子为 $2i$，右孩子为 $2i+1$，双亲为 $\lfloor i/2 \rfloor$。
- **缺点**：对于一般的二叉树（尤其是单支树），为了维持数组下标的逻辑关系，需要存储大量的空结点，造成空间浪费。
- **适用场景**：最适合存储完全二叉树或满二叉树。

## 2. 链式存储结构
二叉树通常采用链式存储，每个结点包含数据域和指针域。

### 结点的结构
```cpp
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild; // 左右孩子指针
} BiTNode, *BiTree;
```

### 存储性质
1. 在含有 $n$ 个结点的二叉链表中，含有 **$n+1$** 个空指针域。
    - *证明*：$n$ 个结点共有 $2n$ 个指针域；$n$ 个结点除了根以外都有一个指针指向它，故有效指针（非空）数为 $n-1$。空指针数 = $2n - (n - 1) = n + 1$。
2. 如果需要频繁访问结点的双亲，可以增加一个 `parent` 指针，形成**三叉链表**。
