# 7.2 顺序查找和折半查找

### 1. 顺序查找 (Linear Search)
从表的一端开始，逐个进行比对。
- **ASL (成功)**：$(n+1)/2$
- **ASL (失败)**：$n+1$
- **时间复杂度**：$O(n)$
- **哨兵技巧**：在数组 0 号位存待查元素，可省去判断边界的开销。

### 2. 折半查找 (Binary Search)
**前提**：必须是有序的顺序表。
- **算法思想**：取中间位置 $mid = (low + high)/2$。
- **ASL**：$\approx \log_2(n+1) - 1$。
- **时间复杂度**：$O(\log_2 n)$。
- **判定树**：折半查找的过程可以用平衡二叉树描述，树的高度即为最大查找次数。

### 3. 分块查找 (Index Search)
将表分为若干块，块内无序，块间有序。
- 先查找索引表（折半或顺序），再在块内顺序查找。
- **ASL**：$L_i + L_s$（索引查找 + 块内查找）。
