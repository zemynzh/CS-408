# ds-06-05: 拓扑排序与关键路径

## 1. AOV 网与拓扑排序
**AOV 网 (Activity On Vertex)**：用顶点表示活动，边表示优先关系的 DAG 图。

### (1) 拓扑排序算法
1. 选择一个入度为 0 的顶点并输出。
2. 删除该顶点及所有以它为尾的弧。
3. 重复 1-2，直至图中无入度为 0 的点。
- **结果**：若输出结点数小于 $n$，说明图中有环。

## 2. AOE 网与关键路径
**AOE 网 (Activity On Edge)**：用边表示活动，权值表示活动持续时间，顶点表示“事件”。

### (1) 四个核心参数
1. **事件 $v_k$ 的最早发生时间 $ve(k)$**：从源点到 $v_k$ 的最长路径。
   $ve(k) = \max\{ve(j) + w(j, k)\}$
2. **事件 $v_k$ 的最迟发生时间 $vl(k)$**：不影响汇点在 $ve(n)$ 完成的前提下，$v_k$ 必须发生的时间。
   $vl(k) = \min\{vl(j) - w(k, j)\}$
3. **活动 $a_i$ 的最早开始时间 $e(i)$**：弧 $\langle v_k, v_j \rangle$ 对应活动的 $e(i) = ve(k)$。
4. **活动 $a_i$ 的最迟开始时间 $l(i)$**：$l(i) = vl(j) - w(k, j)$。

### (2) 关键路径
- **时间余量 $d(i) = l(i) - e(i)$**：$d(i) = 0$ 的活动称为**关键活动**。
- **关键路径**：从源点到汇点全由关键活动构成的路径。
- **特性**：
    - 关键路径是网中最长的路径。
    - 缩短关键活动时间可以缩短工期，但缩短过头可能使原本的非关键路径变成关键路径。
    - 关键路径可能不止一条。
