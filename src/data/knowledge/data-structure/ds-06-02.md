# ds-06-02: 图的存储及基本操作

## 1. 邻接矩阵 (Adjacency Matrix)
使用一个一维数组存储顶点信息，一个二维数组存储边的信息。
- **结构**：`A[i][j] = 1` 表示有边，`0` 表示无边（网中用 $\infty$）。
- **特点**：
    - 空间复杂度 $O(n^2)$。
    - 适合稠密图。
    - 无向图的邻接矩阵是对称的。
    - 容易判断两个顶点间是否有边。

## 2. 邻接表 (Adjacency List)
为每个顶点建立一个单链表，存储所有与该顶点关联的边。
- **结构**：顶点表（存储数据和第一个边表结点指针） + 边表（存储邻接点下标和下一个指针）。
- **特点**：
    - 空间复杂度 $O(n+e)$（无向图为 $O(n+2e)$）。
    - 适合稀疏图。
    - 找邻接点快，但判断两点是否有边较慢。

## 3. 十字链表 (Cross List) —— 有向图专用
- 结合邻接表和逆邻接表。
- 容易找到以顶点 $v$ 为头的弧，也容易找到以 $v$ 为尾的弧。

## 4. 邻接多重表 (Adjacency Multilist) —— 无向图专用
- 解决邻接表中一条边在两个链表中存储两次，导致删除边操作麻烦的问题。

## 5. 基本操作
- `Adjacent(G, x, y)`：判断是否有边。
- `Neighbors(G, x)`：列出所有邻接点。
- `InsertVertex(G, x)`、`DeleteVertex(G, x)`。
- `FirstNeighbor(G, x)`：找第一个邻接点。
- `NextNeighbor(G, x, y)`：找 $x$ 相对于 $y$ 的下一个邻接点。
