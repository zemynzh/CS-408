# 4.2 串的模式匹配

### 1. 朴素模式匹配算法
从主串的第一个字符开始，依次与模式串进行匹配。
- **最坏时间复杂度**：$O(n \times m)$，$n, m$ 分别为主串和模式串长度。

### 2. KMP 算法
当发生不匹配时，主串指针 $i$ **不回溯**，利用已经匹配的部分信息，将模式串向后滑动到合适位置。
- **时间复杂度**：$O(n + m)$。
- **核心：next 数组**
  - `next[j]` 的含义：模式串中第 $j$ 个字符发生不匹配时，应回退到的位置。
  - 定义：前 $j-1$ 个字符构成的子串中，**最长相等前后缀**的长度。

### 3. next 数组的手算步骤 (常考)
1. `next[1] = 0` (有的教材设为 -1)。
2. `next[2] = 1` (有的教材设为 0)。
3. 其他位置：找该位置之前的子串中，最长的相同前缀和后缀，长度加 1。

### 4. KMP 改进：nextval 数组
为了解决如 `aaaaa` 这种模式串在匹配失败时频繁无效比对的问题。
- 若 `p[j] == p[next[j]]`，则 `nextval[j] = nextval[next[j]]`。

> **技巧**：408 考题中需注意 `next` 数组下标是从 0 还是 1 开始。
