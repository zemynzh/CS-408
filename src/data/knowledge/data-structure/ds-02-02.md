# 2.2 顺序表 (Sequential List)

### 1. 定义
用一组**地址连续**的存储单元依次存储线性表的数据元素。逻辑上相邻的两个元素在物理位置上也相邻。

### 2. 特点
- **随机存取**：可以在 $O(1)$ 时间内找到第 $i$ 个元素。
- **存储密度高**：每个结点只存储数据元素本身。
- **插入和删除不便**：需要移动大量元素。

### 3. 重要操作的复杂度
- **插入操作**：平均移动次数为 $n/2$，时间复杂度为 $O(n)$。
- **删除操作**：平均移动次数为 $(n-1)/2$，时间复杂度为 $O(n)$。
- **按值查找 (LocateElem)**：平均比较次数为 $(n+1)/2$，时间复杂度为 $O(n)$。
- **按位查找 (GetElem)**：时间复杂度为 $O(1)$。

### 4. 存储空间分配
- **静态分配**：使用定长数组，空间开辟在栈上，大小不可改变。
- **动态分配**：使用 `malloc` 或 `new`，空间开辟在堆上，可扩容，但仍需移动元素。

> **公式提示**：
> 顺序表中第 $i$ 个元素的地址计算公式：
> $Loc(a_i) = Loc(a_1) + (i - 1) \times sizeof(ElemType)$
