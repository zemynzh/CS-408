# 1.2 算法与算法评价

### 1. 算法的基本概念
- **定义**：算法是对特定问题求解步骤的一种描述。
- **算法 vs 程序**：算法是解决问题的逻辑，程序是用某种语言对算法的具体实现。程序不一定满足有穷性（如操作系统），但算法**必须满足有穷性**。

### 2. 算法的五个特性 (必备常考)
1. **有穷性**：算法必须在执行有穷步之后结束。
2. **确定性**：算法中每条指令必须有确切含义，无二义性。
3. **可行性**：算法中描述的操作都可以通过基本运算有限次实现。
4. **输入**：有零个或多个输入。
5. **输出**：有一个或多个输出。

### 3. 算法效率的度量

#### (1) 时间复杂度 ($T(n)$)
- **定义**：算法执行时间的增长率。通常用大 $O$ 表示法。
- **大 $O$ 表示法**：描述算法时间规模的上界。
- **考纲常见阶数对比**：
  $O(1) < O(\log_2 n) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)$
- **三种情况**：
  - **最坏时间复杂度**：考纲及实际分析的主流，保证算法运行时间不会更久。
  - **平均时间复杂度**：所有输入等概率情况下的期望运行时间。
  - **最好时间复杂度**：通常参考价值较小。

#### (2) 空间复杂度 ($S(n)$)
- **定义**：算法运行过程中所消耗的**辅助存储空间**。
- **原地工作 (In-place)**：辅助空间复杂度为 $O(1)$，即算法所需的额外空间不随问题规模 $n$ 的增大而增大。

### 4. 算法评价标准
- **正确性**：能正确解决问题。
- **可读性**：便于理解和交流。
- **健壮性**：输入非法数据时能做出反应而非崩溃。
- **高效性与低存储量**：时间快、省内存。
