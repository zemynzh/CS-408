# ds-05-05: 树和森林

## 1. 树的存储结构
### (1) 双亲表示法
- 采用顺序存储，每个结点记录其双亲的下标。
- 优点：找双亲极快；缺点：找孩子需要遍历。

### (2) 孩子表示法
- 每个结点的孩子用单链表链接。
- 优点：找孩子快；缺点：找双亲难。

### (3) 孩子兄弟表示法（二叉链表表示法）
- `*fchild` 指向第一个孩子；`*nsibling` 指向下一个兄弟。
- **核心价值**：它是树、森林转化为二叉树的物理基础。

## 2. 树、森林与二叉树的转换
### (1) 树转换为二叉树
- “左孩子右兄弟”原则：根的第一个孩子作为左孩子，该孩子的兄弟作为其右孩子。

### (2) 森林转换为二叉树
- 将每棵树转换为二叉树，然后将第二棵树作为第一棵树根的右子树，第三棵作为第二棵的右子树，依此类推。

### (3) 二叉树还原为森林
- 若二叉树根有右子树，则右子树部分是森林的其余树。

## 3. 树和森林的遍历
| 树 | 森林 | 二叉树 |
| :--- | :--- | :--- |
| 先根遍历 | 先序遍历 | 先序遍历 |
| 后根遍历 | 中序遍历 | 中序遍历 |

**注意**：树的后根遍历对应其转化后二叉树的中序遍历。
