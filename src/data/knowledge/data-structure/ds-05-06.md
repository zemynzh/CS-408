# ds-05-06: 哈夫曼树与哈夫曼编码

## 1. 基本概念
- **路径长度**：路径上边的数目。
- **结点的带权路径长度 (WPL)**：从树根到该结点之间的路径长度与该结点权值的乘积。
- **树的带权路径长度 (WPL)**：树中所有**叶子结点**的带权路径长度之和：
  $$WPL = \sum_{i=1}^n w_i l_i$$
- **哈夫曼树**：在含有 $n$ 个带权叶子结点的二叉树中，WPL 最小的二叉树（又称最优二叉树）。

## 2. 哈夫曼树的构造算法
1. 给定 $n$ 个权值 $\{w_1, w_2, \dots, w_n\}$，构造 $n$ 棵只有根结点的森林。
2. 在森林中选取两棵根结点**权值最小**的树作为左右子树合并，新树根权值为两者之和。
3. 从森林中删除选取的两棵树，加入新树。
4. 重复 2-3 步，直到森林中只剩一棵树。

**特点**：
- 哈夫曼树中不存在度为 1 的结点。
- $n$ 个叶子结点的哈夫曼树总结点数为 $2n-1$。

## 3. 哈夫曼编码
- **前缀编码**：没有一个编码是另一个编码的前缀。可以保证译码无二义性。
- **构造**：在哈夫曼树中，约定左分支为 0，右分支为 1。从根到叶子的路径上的 0/1 序列即为该字符的哈夫曼编码。
- **作用**：数据压缩，使得平均编码长度最短。
